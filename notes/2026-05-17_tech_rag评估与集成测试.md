## RAG 评估基线

### 评估指标（自实现，ETL 对账风格）

| 指标 | 公式 | 含义 | 目标 | ETL 类比 |
|------|------|------|------|---------|
| Recall@5 | 命中相关文档 / 总相关文档 | 召回率 | ≥80% | 数据完整率 |
| Precision@5 | 命中相关文档 / 返回的5个 | 精确率 | ≥60% | 数据准确率 |
| MRR | 1/第一个相关结果排名 | 首位命中率 | ≥0.70 | 关键数据到达率 |
| Answer Relevance | 答案包含标准答案关键词比例 | 回答准确度 | ≥80% | 需求匹配度 |

### 评估数据集

- **3 个测试文档**：公司考勤制度、产品手册、数据库配置指南（`data/test_docs/`）
- **10 个测试问题**：7 个事实查询 + 1 个推理分析 + 1 个无答案 + 1 个 AgentFlow 相关问题
- **独立向量库**：`chroma_data_eval/`（不污染主知识库）

### 运行方式

```bash
python eval/rag_eval.py
```

### 状态

评估脚本已完成但受 Windows 页面文件限制（WinError 1455），需增加虚拟内存后运行。
