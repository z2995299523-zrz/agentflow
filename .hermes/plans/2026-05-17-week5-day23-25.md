## Task 1: BGE 模型预热（FastAPI lifespan）

**目标:** 把 BGE 模型加载从"第一次请求等 15 秒"改为"启动时预热一次"

**文件:**
- 修改: `main.py` — 加入 lifespan 事件
- 修改: `graph/nodes.py` — 支持外部注入 vectorstore / sql_agent

**具体改动:**

```python
# main.py — 新增 lifespan
from contextlib import asynccontextmanager

@asynccontextmanager
async def lifespan(app: FastAPI):
    # 启动预热
    from rag.vectorstore import load_vectorstore
    from sql.agent import SQLQueryAgent
    app.state.vectorstore = load_vectorstore()
    app.state.sql_agent = SQLQueryAgent()
    
    # 注入到 graph 节点
    from graph.nodes import init_vectorstore, init_sql_agent
    init_vectorstore(app.state.vectorstore)
    init_sql_agent(app.state.sql_agent)
    
    yield  # 服务运行
    # 清理（如有需要）

app = FastAPI(lifespan=lifespan)
```

```python
# graph/nodes.py — 新增注入函数
_vectorstore = None
_sql_agent = None

def init_vectorstore(vs):
    global _vectorstore
    _vectorstore = vs

def init_sql_agent(agent):
    global _sql_agent
    _sql_agent = agent

def rag_node(state):
    global _vectorstore
    if _vectorstore is None:  # fallback: 懒加载
        from rag.vectorstore import load_vectorstore
        _vectorstore = load_vectorstore()
    result = ask(question, vectorstore=_vectorstore)
    ...

def sql_node(state):
    global _sql_agent
    if _sql_agent is None:  # fallback: 懒加载
        from sql.agent import SQLQueryAgent
        _sql_agent = SQLQueryAgent()
    result = _sql_agent.query(question)
    ...
```

**验收:** main.py 重启后，直接调 RAG API 不需要等 15 秒

## Task 2: Mixed 意图双路并行（RAG → SQL → 汇总）

**目标:** 当 supervisor 识别为 "mixed" 时，先查文档提取关键条件，再查数据库验证

**文件:**
- 修改: `graph/workflow.py` — 新增 mixed 路由逻辑
- 修改: `graph/supervisor.py` — route_by_intent 不变（mixed → "rag" 已正确）

**图结构调整:**

```
supervisor → [rag | sql | fallback]
rag → route_after_rag →
    intent=mixed → sql → finalize
    intent=rag   → finalize
```

**新增函数:**

```python
def route_after_rag(state: AgentState) -> str:
    """RAG 后路由：mixed 意图需要继续走到 SQL"""
    if state.get("intent") == "mixed":
        return "sql"
    return "finalize"
```

**图连线修改:**

```python
# 改：RAG 节点后加条件路由
graph.add_conditional_edges("rag", route_after_rag, {
    "sql": "sql",
    "finalize": "finalize",
})
```

**验收:** `{"question": "根据产品手册的标准，分析哪些产品达标", "intent": "mixed"}` → RAG 结果 → SQL 结果 → final_answer 包含两者

def build_workflow(enable_memory: bool = True):
    graph = StateGraph(AgentState)

## Task 4: LangGraph 评估基线

**目标:** 建立 LangGraph 工作流的质量评估体系

**文件:**

**评估指标:**

| 指标 | 衡量 | 目标 | 方法 |
|------|------|------|------|
| 路由准确率 | Supervisor 分类正确率 | ≥90% | 人工标注 10 题 × 自动对比 |
| 端到端成功率 | workflow 返回有效 final_answer | ≥80% | 10 题 × invoke |
| 回答完整性 | mixed 意图下同时含 RAG+SQL 结果 | ≥80% | 检查 final_answer 结构 |
| LLM-as-Judge 评分 | DeepSeek 评分（1-5） | ≥3.5 | 自动评估 |

**评估脚本核心逻辑:**

```python
def evaluate_routing(questions, workflow):
    correct = 0
    for q in questions:
        result = supervisor_node({"question": q["question"], ...})
        if result["intent"] == q["expected_intent"]:
            correct += 1
    return correct / len(questions)

def evaluate_e2e(questions, workflow):
    success = 0
    for q in questions:
        result = workflow.invoke({"question": q["question"]})
        if len(result.get("final_answer", "")) >= q["min_length"]:
            success += 1
    return success / len(questions)
```
